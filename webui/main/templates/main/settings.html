Dynamic DNS

Makes your server reachable under a fixed hostname via deSEC, even if your ISP changes your public IP. The DDNS update runs every 5 minutes via cron.

Required for any service that needs to be accessed from the internet (web interface, Authelia, etc.).

Mailserver (SMTP)

Configures an outgoing SMTP relay for Authelia email-based verification (2FA codes, password resets, login notifications).

SMTP settings are auto-detected from your email domain (MX records / autoconfig). Falls back to manual entry if needed.

Configure
Auth

Enables Authelia as a forward-auth middleware behind Traefik. All web services then require login against the local LDAP directory, with optional 2-Factor Authentication.

Users and groups are managed under the respective menu items.

Port Forwarding Guide

For the web interface, Authelia, and all other services behind Traefik to be reachable from the internet, your router must forward TCP ports 80 (HTTP) and 443 (HTTPS) to the internal IP of the Symbios server.

IPv4
  1. Assign your Symbios server a static local IP (e.g. 192.168.1.100) via your router's DHCP reservation.
  2. In your router's Port Forwarding / Virtual Server section, create two rules:
    • TCP 80 → 192.168.1.100:80
    • TCP 443 → 192.168.1.100:443
IPv4 limitation: Many consumer ISPs use Carrier-Grade NAT (CG-NAT) or DS-Lite, making IPv4 port forwarding impossible from the internet. If you have a public IPv4 but forwarding still fails, your ISP may block ports 80/443 — a business plan or VPN tunnel might be required.
IPv6

With IPv6, every device typically gets a global, publicly routable address — no NAT involved. No port forwarding is needed if your firewall permits incoming TCP 80/443.

  1. Enable IPv6 on your router. Verify on the DDNS page that your server has a global IPv6 address (starts with 2001: / 2a0: / 2a01:…).
  2. In your router's Firewall / Traffic Rules section, allow incoming TCP traffic on ports 80 and 443 to your server's IPv6 address.
  3. On the DDNS page, select "IPv6 only" (or "Dual-Stack" if you also have working public IPv4).
Pro tip: If you have only IPv6 (CG-NAT for IPv4), choose IPv6 only — the server stays reachable via IPv6 while the IPv4 A record is automatically removed from deSEC.
