About

deSEC is a free Dynamic DNS provider. Your server periodically (cron, every 30 min) updates the DNS A/AAAA records for your hostname so it always points to your current public IP — even when your ISP changes it.

  • Get an API Token at desec.io
  • Hostname: myserver.dedyn.io (subdomain only)
  • Domain is auto-created if it doesn't exist yet
  • DDNS update runs every 5 minutes via cron
  • Updates are logged in System Logs
Port Forwarding

For the web interface and Authelia to work from the internet, your router must forward ports 80 (HTTP) and 443 (HTTPS) to the internal IP of the Symbios host.

IPv4
  1. Assign your Symbios server a static local IP (e.g. 192.168.1.100) via your router's DHCP reservation.
  2. In your router's Port Forwarding / Virtual Server section, create two rules:
    • TCP 80 → 192.168.1.100:80
    • TCP 443 → 192.168.1.100:443
Important: Many consumer ISPs use Carrier-Grade NAT (CG-NAT) or DS-Lite, which makes IPv4 port forwarding impossible. In that case you must use IPv6 (see below). If you have a public IPv4 and forwarding still doesn't work, your ISP may block ports 80/443 — a business plan or VPN tunnel may be required.
IPv6

With IPv6, every device typically has a global, publicly routable address — no NAT involved. No port forwarding is needed if your firewall allows incoming connections.

  1. Enable IPv6 on your router and confirm your server has a 2001: / 2a0: / 2a01:… address (check Current IPs box on this page).
  2. In your router's Firewall / Traffic Rules section, allow incoming TCP traffic on ports 80 and 443 to your server's IPv6 address or use a firewall rule that permits established/related traffic.
  3. Select "IPv6 only" (or "Dual-Stack" if you also have working public IPv4) in the IPv6 mode dropdown above.
Pro tip: If you have both IPv4 behind CG-NAT and working IPv6, choose IPv6 only — the server will be reachable via IPv6 while the IPv4 A record is automatically removed from deSEC.
